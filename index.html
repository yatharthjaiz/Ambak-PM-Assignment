Part One

Find One Broken Thing

On WhatsApp, users frequently miss important messages from low-frequency but high-priority contacts — like urgent family updates, work messages, or appointment reminders — because the chat list sorts conversations strictly by recency, allowing high-volume groups and broadcast lists to push critical chats out of view; there is no visual distinction for important messages within chats; and the current limit of only 3 pinned chats is insufficient for users managing multiple important relationships across personal, professional, and service-oriented communications.

Real User Scenarios

👨‍👩‍👧‍👦
1 — Family Group Spam
Large family group (50+ members) shares "Good morning" images, jokes & festival greetings — 20–30 messages/hour, every day.
Result: Mother's urgent message about father's hospitalization at 10:03 AM is buried under 40+ new messages by 10:15 AM. Seen only after searching 2 hrs later — a medical emergency delayed.
🎓
2 — Academic Deadline
College course group (120 students) mixes assignment doubts, memes, event promotions. Professors post urgent updates — deadline changes, class cancellations.
Result: Professor's venue change for a final presentation is lost between meme forwards. User arrives at old location 30 min late → embarrassment & possible grade penalty.
🔐
3 — Payment Link Buried
User is waiting for a time-sensitive payment link. Active chats include a friends group, work project group, and a shopping-app broadcast list sending daily deals.
Result: The link arrives but is pushed below the fold within 90 seconds. User requests a new link (also buried), eventually abandons the transaction after 5 minutes of frustration.
📋
4 — Broadcast-List Blind Spot
Subscribed to broadcasts: doctor's office, landlord, and govt. service. These lists only push outward — you can't reply or see other recipients. Rarely opened proactively.
Result: "Your blood test results are ready — collect today before 4 PM." Sits unseen. User misses deadline → repeat test & delayed treatment.

Root Cause — The 5 Whys

Why #1
Why does WhatsApp sort chats strictly by recency? → Because the product assumes the most recently active conversation is likely the most relevant at the moment — matching real-time communication expectations (SMS, iMessage, Telegram).
Why #2
Why doesn't WhatsApp implement smarter sorting? → Because algorithmically determining "importance" is inherently error-prone and risky for a communication tool where missing a truly urgent message could have serious consequences.
Why #3
Why is algorithmic importance considered too risky? → Because WhatsApp's global success relies on being perceived as 100% reliable — users must trust that if someone messages them, they will see it promptly unless they actively ignore it.
Why #4
Why prioritize reliability perception over smart features? → Because WhatsApp serves an extremely diverse global user base — users with low digital literacy, older users, users in poor-connectivity regions, and people who rely on WhatsApp as their only internet service.
🏁 Fundamental Constraint — Why #5

A deliberate product trade-off: WhatsApp chooses universal accessibility and perceived reliability over advanced message prioritization to maintain its position as the simplest, most dependable global communication tool — equally effective for a rural farmer in Uttar Pradesh and a stock trader in Mumbai. This is not a "PM hasn't gotten to it yet" problem. It is a conscious strategic choice rooted in WhatsApp's mission and user-base constraints.

Part Two

Propose a Fix

Smart Color Highlighting 🎨

Users long-press any chat or message → "Highlight" and select from a simple palette of custom colors (Red, Gold, Green, Purple). The entire chat row in the chat list gets a subtle colored background tint and edge strip. A per-chat filter lets users quickly surface only highlighted messages. Cloud-synced tags — color states are saved on the backend to ensure consistency across linked devices (Web, Desktop, Mobile). Upgrades existing message infrastructure with highly scannable visual cues.

What Exactly Changes

👆

Long-Press to Highlight

New "Highlight" option in the long-press menu — allows users to assign a custom color (🔴 🟡 🟢 🟣) to specific messages or entire contacts. Familiar UX pattern.

🎨

Color-Coded Chat List

Chats containing unread highlighted messages feature a subtle color strip and gradient background tint, making them instantly unmissable when scanning down the chat list.

🔍

Per-Chat Color Filter

Swipe-left → Filter → "Show Highlighted Messages." Temporarily surfaces all custom-colored messages in a conversation. Defaults off; fully optional.

☁️

Cloud-Synced Highlights

Color tags are securely synced to the cloud alongside message IDs. This ensures that when a user highlights a message on their phone, it instantly reflects on WhatsApp Web or their linked tablet.

∞

No Artificial Limits

Unlike the 3-pin cap, users can highlight as many items as needed — essential for multi-step transactions, tracking work updates, and managing family emergencies simultaneously.

💡

One-Time Tooltip

"Long-press a message or contact → Highlight to assign a custom color to urgent links, reminders, or key info." Single nudge, dismissible.

Design Decisions Explained

🎨
Edge-to-edge Highlighting (Not just a dot)

A small flag dot can still be missed. By applying a subtle gradient background tint and a bold left-border strip to the entire chat item, it utilizes peripheral vision. Users can scan 50 chats and instantly spot the "Red" one.

🔤
"Highlight" — not "Flag" or "Star"

"Star" already means "bookmark for later" (recipes, links). "Flag" implies reporting spam. "Highlight" is universally understood as calling attention to something important, accessible to users of all tech literacy levels.

⚡
No chat-list sort change — deliberately

This fix does NOT reorder the chat list. Changing sort order is the core trade-off WhatsApp refuses to make (the root of Why #5). This solution works within that constraint — adding a highly visible signal without touching recency-first ordering.

📱
Why not a global "Important" tab?

A global tab requires heavy backend indexing (cross-chat query) and complex server-side state. Per-chat color filtering remains lightweight — leveraging basic cloud-synced flags. Serves 80% of the need at ~10% of the engineering cost.

Rejected Alternatives

Rejected

Alternative 1 — Dynamic Priority Pinning

  • Increase pin limit to unlimited, self-ordered by recent activity within the pinned set.
  • Still manual and incomplete — doesn't help with important messages inside a chat (e.g., a payment link buried in a 200-message thread).
  • Risk of clutter — power users could pin dozens of chats, creating a second messy chat list.
  • Only helps at chat-list level; doesn't help with spotting key messages in long-running threads.
Rejected

Alternative 2 — AI-Powered Importance Detection

  • On-device ML to auto-flag urgency using strings like "urgent", "deadline", "payment failed", or contact labels like "Work".
  • False positives/negatives erode trust in WhatsApp's core reliability promise — even one misfiled emergency is unacceptable.
  • Privacy perception risk — users worry about "scanning" even when processing is on-device.
  • Philosophically misaligned — WhatsApp's brand is simplicity and predictability, not opaque automation.

Measurement

What Success Looks Like

⚠️

Note on data: Since this assignment was completed over a weekend, I do not have actual usage data or test results. Instead, below are the possible positive outcomes I would expect to see if this feature was built, and how we could simply measure if the solution is actually working.

🎯
Primary
Expected Result: Fewer Missed Important Messages
If this feature works, users should feel less anxious about missing things. We could verify this later by asking a small group of users a simple question a few weeks after launch: "In the past week, did you miss an important WhatsApp message?"
Why this matters: This directly solves the core problem. App usage numbers are nice, but the real win is if users actually feel like they are missing fewer critical updates (like urgent links, appointments, or work updates).
📈
Adoption
Expected Result: Steady, Selective Usage
We would expect to see users picking custom colors a few times a week, rather than assigning colors to every single chat.
Why this matters: A typical Indian WhatsApp power user might get 2-3 urgent transaction links or work updates a week. If they use the highlight feature just 2-3 times a week, it means they are using it correctly. If they highlight 30 things a week, the chat list becomes a messy rainbow, meaning the feature isn't special anymore.
📉
Proxy
Expected Result: Drop in "Resend Link" Requests
We would look to see if people stop requesting the same payment or login link multiple times within a short 5-minute window.
Why this matters: People usually ask a service to "resend link" because the first link got buried by other chats and they gave up searching for it. If our color highlights work, they will spot the first link immediately, so duplicate requests should naturally drop.
❌

Metric deliberately excluded: "Time taken to open a highlighted message." This sounds good in theory, but it is very complicated to accurately track when someone actually *saw* a message versus when they opened it. The drop in "resend link" requests is a much simpler, more reliable way to see if people are finding their messages faster.

Risks & Trade-offs

Risk
Color Clutter (Rainbow Effect)
Why It Happens
If coloring is too easy, users might assign colors to every single contact, turning their chat list into a visually overwhelming rainbow, diluting the "urgent" signal completely.
Mitigation
Keep background tint gradient very subtle. First-use nudge: "Use highlights sparingly for truly time-sensitive info." Monitor highlight density — if avg highlights/chat >20, add a gentle reminder.
Risk
Simplicity Conflict
Why It Happens
Adding a new menu item and color palette introduces elements users must learn — possibly confusing low-literacy users like Sunita.
Mitigation
Feature lives only in the long-press menu — no permanent persistent UI clutter. WhatsApp has shipped starred messages, disappearing messages, and custom chat wallpapers without eroding its simplicity perception.
Risk
Confusion with Stars
Why It Happens
Users already bookmark messages via stars. May not see the difference or use both inconsistently.
Mitigation
Clear copy: "Highlight" = time-sensitive visual cue; "Star" = bookmarking recipes, memes, links. In-app tooltip on first use. If confusion persists at scale, consider merging concepts in Phase 2.
Risk
Expectation Creep
Why It Happens
Once manual highlighting works, users will demand AI auto-coloring — pulling WhatsApp toward the complexity it strategically avoids.
Mitigation
Accept as a positive demand signal. Keep launch strictly manual. Use adoption data to inform a controlled Phase 2 toggle — only after validating the manual version's safety and retention impact over 90 days.

Part Three

The Honest Part

⚠️

The Single Weakest Claim

Claim I'm least confident in: That users will consistently and correctly use the manual "Highlight" color feature without creating visual clutter.


The solution depends entirely on a behavior change — people must remember to long-press and color-code each important message before they lose track of it. In practice, users may forget, color inconsistently, or just color everything "Red," turning their inbox into a chaotic rainbow that defeats the purpose of the feature.


My confidence here is lower than in the technical feasibility because it hinges on unpredictable human factors, not a deterministic design constraint. I haven't validated this with real users — which is exactly what I'd do first with more time.

🔬

If I Had One More Week

I would conduct generative user interviews with 5–6 WhatsApp power users:

  • A college student in multiple course groups
  • A professional in work and family groups
  • A parent who relies on broadcast lists

Instead of jumping straight to building or prototyping, I would ask them to open their actual WhatsApp chat lists and walk me through their recent history. I'd ask them to point out specific messages they missed or almost missed.


My goal: Validate if asking users to manually highlight and color-code messages creates too much friction in a real-world context. Pure reasoning can't tell me if a user in a hurry will actually remember to use this feature.

Part Four (Bonus)

Product Spec

a. Goal / Problem Statement

Enable users to surface low-frequency but high-priority messages (login links, appointment reminders, work updates) that get buried in recency-sorted chat lists, without compromising WhatsApp’s core simplicity and reliability.

b. Proposed Solution

  • Where: Long-press menu on any message or chat, and the main Chat List view.
  • What changes (UI): Add a "Highlight" option with a 4-color palette to the long-press menu. Apply a colored edge strip and subtle gradient background to highlighted chats in the main list. Add a "Show Highlighted Messages" swipe filter.
  • What changes (Backend): Store color tags via local SQLite and securely sync minimal metadata (Message ID + Color Hex) via cloud to maintain highlight state across linked devices.

c. User Stories

As a...
Power user in many groups
I want to...
Highlight a critical login link or reminder message
So that I...
Can spot it instantly via the custom color strip in the chat list.
As a...
User who misses appointments
I want to...
Color code a doctor’s reminder as important
So that I...
Reduce the chance of missing the appointment and avoid rescheduling costs.
As a...
Multi-device user
I want to...
Have my highlights synced to the cloud
So that I...
Can see the same color-coded priority messages on my phone, tablet, and WhatsApp Web.

d. Success Metrics

Primary & Adoption

  • Missed Messages: Reduction in self-reported missed urgent messages (via post-launch micro-survey).
  • Usage Depth: Steady, selective usage of 1-5 highlights per week per user (avoids "color clutter").

Proxy Metric

  • Duplicate Links: Measurable drop in duplicate "resend link" requests within a 5-minute window for authentication and payment services.

e. Open Questions & Risks

User Adoption

Will users actually remember to manually color-code messages in the moment?

Mitigation: Implement a subtle UI onboarding tip and monitor highlight density metrics early in the rollout.

False Sense of Automation

Users may expect AI auto-tagging once they see the feature exists.

Mitigation: Launch strictly as a manual feature to set baseline expectations; collect demand data for a possible Phase 2.
